Please create an initial `CLAUDE_UPDATES.md` file for this repository.

Before creating the file, review:

* `README.md`
* `NEWS.md`
* `CLAUDE.md`
* `CLAUDE_ARCHITECTURE.md`
* Recent git history

Goals:

* Create a Claude-maintained summary of recent project evolution.
* Focus on changes that help future Claude sessions understand how the project has evolved.
* Do not duplicate git history.
* Do not duplicate `NEWS.md`.
* Focus on architectural, operational, deployment, dependency, and workflow changes.
* Ignore routine bug fixes, formatting changes, and minor refactoring unless they significantly affected the system.

The file should:

* Be concise.
* Be human-readable.
* Be useful to both developers and future Claude sessions.
* Include a short explanation of the purpose of the file.
* Include the date of creation.
* Summarize the most important project changes from the recent history of the repository.
* Identify any changes that materially affected the current architecture described in `CLAUDE_ARCHITECTURE.md`.
* Be organized in reverse-chronological order with the most recent updates at the top.

When complete, create `CLAUDE_UPDATES.md` and then explain your rationale for the structure you selected.
